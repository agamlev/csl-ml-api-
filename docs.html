CSL Won Probability API

Predict the likelihood of closing a deal based on 6 customer signals

ML Powered

Endpoint

POST https://csl-ml-api.onrender.com/predict

Send a JSON body with 6 customer parameters. Receive a win probability percentage in return.

Request Parameters

Parameter Type Valid Values Description
customer_health_score Integer 0 – 100 Overall customer health score
customer_plan String Basic / Pro / Premium / Enterprise Customer subscription plan
usage_volume Integer 0 – 2500 Product usage volume
restricted_feature_attempt Integer 0 – 6 Attempts to access restricted features
feature_related_inquiry Integer 0 – 5 Number of feature-related inquiries
csat_score Float 1.0 – 5.0 Customer satisfaction score

How to Use in Make

  1. Add an HTTP → Make a Request module to your scenario.

  2. Set the URL to https://csl-ml-api.onrender.com/predict and Method to POST.

  3. Set Body content type to application/json and Body input method to JSON string.

  4. Paste the JSON body with the 6 parameters — map each field from your previous modules (CRM, HubSpot, etc.).

  5. Enable Parse response → Yes so Make can read won_probability_% as a variable in the next step.

  6. Add a Router after the HTTP module — route based on the probability value to trigger different actions (Slack alert, CRM update, email, etc.).
